# MM
Miracle Morning

## Local Run (CORS-safe)

Do not open `index.html` directly with `file://`.
Run a local HTTP server, then open the page in a browser.

### Option 1: PowerShell script (recommended)

```powershell
cd C:\Users\abcra\OneDrive\Coding\MM
powershell -ExecutionPolicy Bypass -File .\scripts\serve.ps1
```

Default URL: `http://localhost:8080`

### Option 2: Batch file

Double-click `scripts\serve.bat`, or run:

```powershell
cd C:\Users\abcra\OneDrive\Coding\MM
.\scripts\serve.bat
```

### Stop server

Press `Ctrl + C` in the terminal where the server is running.

## GitHub Pages + Actions (No API key exposure)

This project is configured to keep API keys out of `index.html`.
Frontend reads only:
- `data/market.json`
- `data/news.json`

Data files are generated by GitHub Actions workflow:
- `.github/workflows/update-data.yml`
- `scripts/update_data.py`

### One-time setup (GitHub)

1. Push this repo to GitHub.
2. In GitHub repo, go to `Settings > Secrets and variables > Actions`.
3. Add repository secrets:
   - `NEWS_API_KEY`
   - `TWELVEDATA_API_KEY`
4. Go to `Actions`, run `Update Dashboard Data` once (`Run workflow`).
5. Go to `Settings > Pages`, set:
   - Source: `Deploy from a branch`
   - Branch: `master` (or `main`) / root
6. Open your Pages URL and verify `data/*.json` is populated.

### Update cycle

- Workflow runs every hour (`15 * * * *`) and on manual dispatch.
- If `data/market.json` or `data/news.json` changed, workflow auto-commits.
